# مشروع Moysser: منصة الربط الموحد لبوابات الدفع (Payment Gateway Aggregator)

## 1. ملخص المشروع (Executive Summary)
مشروع **Moysser** هو عبارة عن حل برمجـي وسيط (Middleware) يهدف إلى سد الفجوة بين منصات التجارة الإلكترونية (مثل Ecwid) وبوابات الدفع المتعددة (مثل Payone, Moyasar, Noon). يوفر النظام واجهة برمجة تطبيقات موحدة (Unified API) تسمح للتجار بقبول المدفوعات من مصادر مختلفة بجهد تقني ضئيل، مع معالجة كافة التعقيدات البرمجية لكل بوابة دفع بشكل مستقل.

## 2. مبررات المشروع (Problem Statement)
يواجه التجار والمطورون في المنطقة العربية عدة تحديات عند ربط بوابات الدفع:
* **تعقيد الربط:** اختلاف البروتوكولات والمعايير بين بوابات الدفع (بعضها حديث وبعضها يعتمد أنظمة قديمة).
* **توافق المنصات:** عدم وجود تطبيقات مباشرة تربط بين منصات مثل Ecwid وبعض بوابات الدفع المحلية مثل Payone.
* **إدارة الأخطاء:** صعوبة تتبع مشاكل الدفع وتنسيق البيانات بين المتجر وبوابة الدفع.

## 3. الحل المقترح (The Solution)
يقدم Moysser منصة مركزية تقوم بـ:
* **Order Mapping:** تحويل بيانات الطلبات من المتاجر تلقائياً إلى التنسيق المطلوب لكل بوابة دفع.
* **Payload Orchestration:** معالجة البيانات الحساسة (مثل العملات والأرقام العشرية) بما يتوافق مع الأنظمة البنكية.
* **Redirect Management:** توفير تجربة مستخدم سلسة لتحويل العميل لصفحة الدفع وإعادته للمتجر.

## 4. الابتكار التقني (Key Innovation: The Payone Breakthrough)
أحد أهم نقاط القوة في هذا المشروع هو حل "معضلة الربط مع Payone"، حيث تمكن النظام من كسر شفرة متطلبات الربط المعقدة من خلال:
* **ISO 682 Standard:** التعامل مع العملة بالترميز الرقمي العالمي.
* **Minor Units Engine:** نظام تحويل المبالغ إلى وحدات صغرى (هلالات) بدقة 100%.
* **Schema Validation:** اكتشاف الاعتمادية المتبادلة بين الحقول الإجبارية (مثل `dynamicFields` و `notificationEmail`) التي نادراً ما يتم توثيقها في الأدلة الرسمية.

## 5. البنية التحتية والتقنيات (Tech Stack)
* **Back-end:** Node.js (Express Framework).
* **Architecture:** Modular Service-Oriented Architecture.
* **Deployment:** Serverless environment (Vercel).
* **Security:** Environment Variables masking, URL SearchParams encoding, and strict Token cleaning.
* **Communication:** RESTful APIs using Native Fetch and URLSearchParams.

## 6. حماية الكود وحقوق الملكية (Intellectual Property)
تم تصميم النظام ليكون ملكية فكرية محمية من خلال:
1. **Separation of Concerns:** فصل اللوجيك الخاص بالربط عن واجهة المستخدم.
2. **Obfuscation Ready:** الكود مهيأ لعملية التشفير البرمجي قبل الإنتاج.
3. **Configuration Isolation:** عزل كافة كلمات السر والمفاتيح في طبقة البيئة (Env), مما يمنع تسريبها حتى في حالة الوصول للكود.

## 7. خارطة الطريق المستقبلية (Future Roadmap)

* **AI Fraud Detection:** دمج أنظمة الذكاء الاصطناعي لاكتشاف عمليات الاحتيال قبل إتمامها.
